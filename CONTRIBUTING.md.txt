```markdown
# 🤝 دليل المساهمين (CONTRIBUTING.md)

شكراً لاهتمامك بمساهمة في هذا المشروع! نقدر جهودك ونهدف لجعل عملية المساهمة سلسة وشفافة.

---

## 📌 كيفية المساهمة

### 1. الإبلاغ عن الأخطاء
- استخدم **قائمة المشكلات (Issues)** للإبلاغ عن:
  - أخطاء في المحتوى
  - روابط مكسورة
  - معلومات غير دقيقة
- قدم تفاصيل واضحة:
  - عنوان وصفي
  - وصف الخطأ
  - خطوات إعادة الإنتاج (إن أمكن)
  - لقطات شاشة (إن لزم الأمر)

### 2. اقتراح تحسينات
- افتح **Issue** لوصف:
  - تحسينات في المحتوى
  - إضافة مواضيع جديدة
  - تحسينات في التنظيم
- اشرح:
  - المشكلة الحالية
  - الحلول المقترحة
  - الفوائد المتوقعة

### 3. تقديم تعديلات
1. **شوك (Fork)** المستودع
2. أنشئ **فرع (Branch)** جديد:
   ```bash
   git checkout -b fix/description-of-change
   ```
3. قم بالتعديلات اللازمة
4. تأكد من:
   - اتباع نفس أسلوب الكتابة
   - الحفاظ على التنسيق (Markdown)
   - عدم كسر الروابط الحالية
5. ارفع التغييرات إلى فرعك
6. افتتح **Pull Request** مع:
   - عنوان واضح
   - وصف للتغييرات
   - مراجع لمشاكل ذات صلة (إن وجدت)

---

## 📝 معايير المحتوى

### 1. أسلوب الكتابة
- **لغة واضحة وبسيطة**: تجنب المصطلحات المعقدة دون شرح
- **تنسيق موحد**:
  - عناوين واضحة (`##`, `###`)
  - قوائم مرقمة/نقطية
  - جداول عند المقارنة
- **أمثلة عملية**: استخدم حالات واقعية من عالم السيارات

### 2. التنسيق الفني
- **كود وبرامج**:
  ```markdown
  ```python
  # مثال كود
  print("Hello World")
  ```
  ```
- **رسوم بيانية**:
  ```markdown
  ```mermaid
  graph LR
    A[البطارية] --> B[المفتاح]
    B --> C[المصباح]
  ```
  ```
- **صور**: وضع في مجلد `assets/` مع اسم وصفي

### 3. المراجع والمصادر
- ذكر المصادر عند الاقتباس:
  ```markdown
  [المصدر: كتاب أساسيات الإلكترونيات، 2023]
  ```
- تجنب الروابط الخارجية إلا إذا كانت ضرورية

## 🔧 بيئة التطوير

### المتطلبات
- محرر نصوص يدعم Markdown (VS Code موصى به)
